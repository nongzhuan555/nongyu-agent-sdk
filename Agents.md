## 2. 代码风格指南
- **命名约定**：
  - 变量、函数、方法：使用 snake_case
  - 类名：使用 PascalCase
  - 常量：使用全大写和下划线
  - 文件和目录：使用小写和下划线
- **代码格式**：
  - 缩进：使用 4 个空格
  - 行长度：每行不超过 80-100 个字符
  - 空行：在函数和类之间使用空行分隔
- **注释规范**：
  - 函数和方法：使用JS的文档注释
  - 复杂逻辑：添加内联注释
  - 避免冗余注释：只注释必要的内容
  - 注释语言：中文

## 3. 版本控制策略与习惯
- **版本控制系统**：使用 Git 作为版本控制系统
- **分支策略**：暂时只有一个分支，即 main 分支，所有开发都在该分支上进行
- **代码提交规范**：
  - 每个提交应包含清晰的提交信息
  - feat: 新增功能或变更
  - fix: 修复 bug 或错误
  - docs: 文档变更
  - test: 测试相关变更
- **注意事项**：代码提交由我本人亲自操作，编码Agent不参与代码提交。
