# 记录本项目开发过程中的一些思考

### 关于工具系统
###### 关于“万能工具”的把控？
类似于bash和本项目中WebFetcher这样的工具，被称之为万能工具或者说通用工具，以bash来说，它能够执行任何命令，而WebFetcher则能够从任意URL获取内容。但我们实际开发Agent的时候，除了这种看似万能的工具，其实也会提供更精细化的工具
比如本项目中的jiaowuNotice工具，它封装了四川农业大学教务网通知的抓取，但实战过程中发现，如果没有约束和引导，在我要求
它回答和四川农业大学通知相关的问题是，它会根据自己训练的数据或者纯粹的编造一个可能有这些信息的URL去获取，而不是调用jiaowuNotice工具从教务网获取。因此，一个反直觉的更佳实践是，工具的描述中不仅要包含工具应该什么情况被调用，也应该告知模型什么情况下不能用这个工具。
